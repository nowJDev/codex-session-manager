## 실패했던 접근과 주의점
- 로컬 `pnpm tauri build`는 처음에 `pnpm`이 PATH에 없어 실패했다.
  - `corepack prepare pnpm@11.8.0 --activate` 후에도 현재 셸 PATH에 shim이 안 잡혔다.
  - 해결: `C:\Program Files\nodejs\node_modules\corepack\shims`를 PATH 앞에 붙여 빌드했다.
- 로컬 Tauri build는 installer 파일 생성 후 updater 서명 단계에서 실패했다.
  - 원인: 로컬 환경에 `TAURI_SIGNING_PRIVATE_KEY`가 없음.
  - GitHub Actions에서는 repo secrets로 서명 및 `latest.json` 생성까지 성공했다.
- `gh` 명령은 기본 repo context가 한때 `glowElephant/claude-session-manager`로 잡혔다.
  - 해결: GitHub Release/Actions 조회와 실행에 `-R nowJDev/codex-session-manager`를 명시했다.
- `v0.5.5` 태그 push만으로 Release workflow가 바로 보이지 않았다.
  - 이전 이력도 `workflow_dispatch` 기반이라 `gh workflow run Release -R nowJDev/codex-session-manager --ref v0.5.5`로 직접 실행했다.
- Vite dev server와 build는 샌드박스에서 `Cannot read directory "../../..": Access is denied`가 날 수 있다.
  - 필요하면 `require_escalated`로 실행해야 한다.
- README 스크린샷 생성 때 임시 `.screenshot-harness.html`을 만들었다가 삭제했다.
  - 이 파일은 커밋 대상에 남기지 않았다.
- 캡처용 Vite node 프로세스만 종료했다.
  - Codex 자체 node 프로세스는 건드리지 않았다.
