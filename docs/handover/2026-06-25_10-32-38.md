## 실패했던 접근과 주의점
- 처음에는 작은 창에서 컬럼 폭을 비례 축소하는 방식으로 수정했다.
  - 결과적으로 정보가 너무 압축되어 읽기 어려운 방향이어서 가로 스크롤 방식으로 변경했다.
- Tauri dev 앱을 `Start-Process -WindowStyle Hidden`으로 띄웠을 때 실제 창 핸들이 잡히지 않았다.
  - 일반 창으로 다시 띄워도 이 환경에서는 Win32 창 핸들이 안정적으로 잡히지 않았다.
  - README 스크린샷은 임시 `.screenshot-harness.html`을 만들어 Chrome headless로 캡처했다.
  - 임시 하네스와 dev 로그 파일은 삭제했다.
- `cmd /c node_modules\.bin\vite build`는 일반 샌드박스에서 상위 경로 읽기 제한 때문에 `Access is denied`가 날 수 있다.
  - 필요하면 `require_escalated`로 실행해야 한다.
- GitHub Release workflow는 이전 이력처럼 태그 push만 믿지 않고 `gh workflow run Release -R nowJDev/codex-session-manager --ref v0.5.6`로 직접 실행했다.
- README 스크린샷은 공개 저장소에 노출되므로 세션 이름/설명 같은 개인 작업 내용이 읽히지 않도록 계속 주의해야 한다.
