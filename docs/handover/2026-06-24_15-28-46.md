- 이름과 설명의 기본값은 저장된 수동 메타데이터, Codex `session_index.jsonl`의 `thread_name`, 실제 첫 사용자 메시지 순서로 정한다.

## 실패했던 접근과 해결

- `codex delete <uuid>`는 비대화형 환경에서 확인을 받을 수 없어 실패했다.
- 해결은 `codex delete --force <uuid>`를 쓰는 방식으로 했다.
- `새 터미널에서 열기`는 명령 계획 자체는 맞았지만 Tauri GUI 프로세스에서 새 콘솔이 보이지 않는 문제가 있었다.
- 해결은 Windows resume spawn에 `CREATE_NEW_CONSOLE` 플래그를 추가하는 방식으로 했다.
- 요약 재생성은 `gpt-5-codex` 모델을 강제해 사용자 계정에서 `model is not supported when using Codex with a ChatGPT account` 오류가 났다.
- 해결은 모델 강제를 제거하고 Codex CLI 기본 모델 설정을 따르게 하는 방식으로 했다.
- `vite build`의 첫 실패는 코드 문제가 아니라 샌드박스 파일 접근 제한이었다.
